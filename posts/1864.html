Spring AOP

1、基本概念

面向切面编程，利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发效率。

通俗解释：不通过修改代码的方式，在主干功能中添加新的功能。

2、底层原理

动态代理：

1、有接口：使用JDK动态代理

就是创建接口实现类代理对象，在代理对象中增强类的方法。

2、没有接口：使用CGLIB动态代理

创建当前类的子类的代理对象，完成类的方法的增强。

3、JDK动态代理

1、使用Proxy类里面的方法创建代理对象

通过”Proxy”类提供的一个newProxyInstance方法用来创建一个对象的代理对象。

static Object newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)

参数：

参数一：类加载器。

参数二：指明生成哪个对象的代理对象，通过接口指定，支持多个接口。

参数三：实现这个接口InvocationHandler，创建代理对象，写增强的方法，指明产生的这个代理对象要做什么事情。

    @Override
    //proxy是代理对象；method是当前被代理对象的方法；args是传入的参数
    public Object invoke (Object proxy, Method method, Object[] args) throws Throwable &#123;

        System.out.println ("方法之前执行。。。"+method.getName()+":传递的参数..."+ Arrays.toString(args));

        //被增强的方法执行
        Object res=method.invoke(obj, args);

        //方法之后
        System.out.println("方法之后执行..."+obj);

        return res;

3、引入AOP相关依赖

4、切入点表达

（1） 切入点表达式的作用：知道对哪个类里面的哪个方法进行增强

（2）语法结构：

​ execution（[权限修饰符].[返回类型].[类全路径].[方法名称]（[参数列表]））

例 1: 对com.dao.BookDao类里面的add进行增强
execution（*com.dao.BookDao.add(..)） // * 指通用

例2：对com.dao.BookDao类里面的所有方法进行增强
Execution（*com.dao.BookDao.*（..））

例3：对com.dao包里所有类，类里面所有方法进行增强
Execution(*com.dao.*.*(..))

3、Spring配置文件中配置切入点

<!--配置aop增强-->
<aop:config>
        <!--切入点-->
        <aop:pointcut id="p" expression="execution(* com.spring5.aop2.Book.buy(..))"></aop:pointcut>

        <!--切入面-->
        <aop:aspect ref="bookProxy">
                <!--增强作用在具体方法上-->
                <aop:before method="before" pointcut-ref="p"></aop:before>
        </aop:aspect>
</aop:config>
