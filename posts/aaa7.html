Xml自动自动装配

例：Emp类中定义Dept类型属性

public class Emp &#123;
    Dept dept;

    public void setDept(Dept dept) &#123;
        this.dept = dept;
    &#125;

    @Override
    public String toString() &#123;
        return "Emp&#123;" +
                "dept=" + dept +
                '&#125;';
    &#125;
&#125;

配置xml文件：

<!--自动装配对象属性-->
<bean id="emp" class="com.spring5.autowire.Emp" autowire="byName">
    <!--autowire="byName"表示自动配置对象属性，不需要再用property等属性引用对象-->
</bean>

<!--但是autowire="byName"被引用的对象其id值要和引用者定义的属性名一样-->
<bean id="dept" class="com.spring5.autowire.Dept">
</bean>

外部属性文件

例：配置数据库信息

  1. 直接配置数据库信息

    （1） 配置德鲁伊连接池

    <!-- 配置连接池 -->
<!-- DruidDataSource dataSource = new DruidDataSource(); -->
<bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
    <!-- dataSource.setDriverClassName("com.mysql.jdbc.Driver"); set方法注入-->
    <property name="driverClassName" value="com.mysql.jdbc.Driver"></property>
    <property name="url" value="jdbc:mysql:localhost:3306/radar"></property>
    <property name="username" value="root"></property>
    <property name="password" value="123456"></property>
</bean>

    ​ (2)引入德鲁伊连接池的jar包

  2. 引入外部的属性文件配置数据库连接池

    (1) 创建外部属性文件，properties格式文件（jdbc.properties），数据库信息

    prop.driverClass=com.mysql.jdbc.Driver
prop,url=jdbc:mysql://localhost:3306/radar
prop.userName=root
prop.password=123456

    (2) 把外部properties属性文件引入到spring配置文件中

    ​ 引入context命名空间

    xmlns:context="http://www.springframework.org/schema/context"
xsi:schemaLocation=" http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd"

    ​ 在spring配置文件中使用标引入外部属性文件

    <context:property-placeholder location="classpath*:jdbc.properties"/>

    ​ 配置连接池

    <bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
 <!-- dataSource.setDriverClassName("com.mysql.jdbc.Driver"); set方法注入-->
    <property name="driverClassName" value="$&#123;prop.driverClass&#125;"></property>
    <property name="url" value="$&#123;prop,url&#125;"></property>
    <property name="username" value="$&#123;prop.userName&#125;"></property>
    <property name="password" value='$&#123;prop.password&#125;'></property>
</bean>

基于注解方式

1.注解

​ （1） 代码的特殊标记

​ （2） 格式：@注解名称（属性名称=属性值，属性名称=属性值）

​ （3） 注解作用在类、方法、属性上面

​ （4） 使用注解的目的是简化xml配置

  1. Spring针对Bean管理中创建对象提供注解

    （1）@Component

    （2）@Service

    （3）@Controller

    （4）@Repository

  2. 基于注解方式实现对象的创建

    第一步 引入依赖jar包

    第二步 开启组件扫描

    <!--引入context命名空间-->
xmlns:context="http://www.springframework.org/schema/context"
xsi:schemaLocation="        http://www.springframework.org/schema/context http://www.springframework.org/schema/beans/spring-context.xsd"
    <!--开启组件扫描-->
<!--base-package是包的位置-->

<!--扫描多个包:
   1.用逗号隔开
   2.扫描上层目录-->

</context:component-scan>
<context:component-scan base-package="com.spring5">
</context:component-scan>

    第三步 创建类，在类上面添加创建对象注解

  1. 开启组件扫描细节配置

    示例1：选择部分扫描

    <context:component-scan base-package="com.spring5" use-default-filters="false">
    <!--context:include-filter,设置扫描哪些内容
    expression表明扫描哪些注解-->
    <context:include-filter type="annotation"
                            expression="org.springframework.stereotype.Controller"/>
</context:component-scan>

    示例2：选择部分不扫描

    <!--扫描所有内容，除了
    context:exclude-filter，设置不扫描的内容-->
<context:component-scan base-package="com.spring5">
    <context:exclude-filter type="annotation"                       expression="org.springframework.stereotype.Controller"/>
</context:component-scan>

基于注解方式属性的注入

（1）@AutoWired 根据属性类型自动装配

​ AutoWired会根据属性的类型，找到其对象，完成注入。

​ 例：

​ 第一步：创建service和dao对象，在service和dao类中创建对象注解

@Service  //对象注解，接口和实现类都要注解
public interface UserDao &#123;
    public void add();
&#125;

@Controller   //对象注解，接口和实现类都要注解
public class UserDaoImpl implements UserDao&#123;
    @Override
    public void add() &#123;
        System.out.println("add dao......");
    &#125;
&#125;
@Service  //对象注解
public class UserService &#123;

    public void add()&#123;
        System.out.println("service add......");
        //userDao.add(); //调用userDao类定义的方法
    &#125;
&#125;

​ 第二步：在service中注入dao对象，在service类添加dao类型属性，在属性上面使用注解

@Service  //对象注解
public class UserService &#123;
    //注解不需要添加set方法，因为已经封装好
    @Autowired  //在属性上面使用注解
    private UserDao userDao;  //在service类添加dao类型属性

    public void add()&#123;
        System.out.println("service add......");
        userDao.add(); //调用userDao类定义的方法
    &#125;
&#125;

​ (2) @Qualifier 根据属性名称进行注入

​ 要和@AutoWired一起用

@Autowired  //在属性上面使用注解
@Qualifier(value="UserDaoImpl1")  //通过属性名称注解（被调用对象注解时的value值）
private UserDao userDao;  //在service类添加dao类型属性

​ 其中@Qualifier的value值要与UserDaoImpl类上注解中的值一致

@Repository (value="UserDaoImpl1")

public class UserDaoImpl implements UserDao&#123;

    @Override
    public void add() &#123;
        System.out.println("add dao......");
    &#125;

&#125;

​ (3) @Resource 根据名称或类型注入

​ 根据类型:

@Resource
private UserDao userDao;

​ 根据名称:

@Resource (name="UserDaoImpl1") //这里是name属性，根据被调用对象注解时的value值
private UserDao userDao;

（4）@Value 注入普通类型属性

@Value(value ="abc")
private String name;
