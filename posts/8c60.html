一、Spring框架概述

1. 概述

  1. Spring是一个轻量级的开源的JavaEE的框架。

  2. 解决企业开发的复杂问题。

  3. 有很多组成部分，两个核心部分：IOC和Aop

    1. IOC：控制反转，把创建对象的过程交给Spring进行管理。

    2. Aop：面向切面，不修改源代码进行功能增强。

    3. 特点：

      1. 方便解耦，简化开发

      2. Aop编程支持

      3. 方便程序测试

      4. 方便集成各种优秀框架

      5. 方便进行事务操作

      6. 降低API开发难度

2. 基本过程

  1. 安装Spring5 Spring.io

  2. 新建项目

  3. 导入相关jar包

  4. 创建普通类普通方法

  5. 创建spring配置文件，在其中配置创建的对象

    1. Spring配置文件使用xml格式

    2）配置对象：

    <bean id="user" class="com.spring5.User"></bean> <!--id是给类起个名字，class是类的路径-->

3. IOC简介

  1. 概念

    控制反转，把创建对象和对象之间的调用过程交给Spring管理。使用IOC的目的是降低耦合度。

    借助于“第三方”实现具有依赖关系的对象之间的解耦，这个“第三方“就是IOC。

    如果对象A依赖于对象B，那么对象A在初始化或者运行到某一点的时候，自己必须主动去创建对象B或者使用已经创建的对象B。无论是创建还是使用对象B，控制权都在自己手上。 软件系统在引入IOC容器之后，这种情形就完全改变了，由于IOC容器的加入，对象A与对象B之间失去了直接联系，所以，当对象A运行到需要对象B的时候，IOC容器会主动创建一个对象B注入到对象A需要的地方。对象A获得依赖对象B的过程,由主动行为变为了被动行为，控制权颠倒过来了，这就是“控制反转”这个名称的由来。

    IOC也叫依赖注入(DI), 所谓依赖注入，就是由IOC容器在运行期间，动态地将某种依赖关系注入到对象之中。所以，依赖注入(DI)和控制反转(IOC)是从不同的角度的描述的同一件事情。

  1. IOC接口

    1） IOC思想基于IOC容器完成，IOC容器底层就是对象工厂

    2）Spring提供IOC容器实现两种方式（两个接口）：

    ​ a. BeanFactory: IOC容器基本实现方式， 是spring的内部使用接口，不提供开发人员使用（加载 配置文件时不会创建对象，在获取（使用）对象时才去创建对象）

    ​ b. ApplicationContext：BeanFactory接口的子接口，提供更多更强大的功能，开发人员使用 （加载配置文件时就会把配置文件中的对象创建）

  1. IOC操作Bean管理

    Bean管理包括两个部分：

    a. Spring创建对象

    b. Spring注入属性

​ Bean管理操作有两种方式：

基于xml配置文件方式实现

<!--配置User对象-->
<bean id="user" class="com.spring5.User"></bean>

a. 在spring配置文件中，使用bean标签，标签里面加入相应的属性，就可以实现对象创建

b. 常用属性：

​ id: 唯一标识

​ class: 类的全路径

​ name: 与id类似，name中可以加特殊符号，id不行

c. 创建对象时，默认也是执行无参构造方法

基于xml方式注入属性

  1. 使用set方法进行注入：

​ 在类中定义参数和set方法

private String bname;
private String bauthor;

//set方法注入
public void setBname(String bname) &#123;
    this.bname = bname;
&#125;

public void setBauthor(String author) &#123;
    this.bauthor = author;
&#125;

​ 在xml文件中配置

<bean id="book" class="com.spring5.Book">
    <!--property是属性配置标签-->
    <property name="bname" value="哈利波特"></property>  <!--这里配置属性时，book类中要定义相应参数的set方法-->
    <property name="bauthor" value="罗琳"></property>
</bean>

​ 测试

ApplicationContext context=new ClassPathXmlApplicationContext("bean1.xml");

Book book=context.getBean("book", Book.class); //第一个参数就是bean标签的id属性值
System.out.println(book);

book.test();

​ 第二种注入方式：有参构造输入

​ (1) 创建类，定义属性，创建属性对应的有参构造方法

public class Orders &#123;
    //属性
    private String oname;
    private String address;

    //有参构造
    public Orders(String oname, String address)&#123;
        this.oname=oname;
        this.address=address;
    &#125;
&#125;

​ (2) 在spring配置文件中配置

<bean id="orders" class="com.spring5.Orders">
    <constructor-arg name="address" value="China"></constructor-arg>
    <constructor-arg index="0" value="evan"></constructor-arg> <!--index指有参构造中的参数索引位置-->
</bean>

注入其他属性

  1. 字面量

    设置空值

    <!--null值-->
<property name="address"><null/></property>

​ 特殊符号

<!--属性包含特殊符号
    1. 把<>进行转义 &lt  &gt
    2. 把带特殊符号内容写到CDATA-->
<property name="address">
    <value><![CDATA[<<南京>>]]></value>
</property>

  1. 外部Bean

    （1）创建两个类service类和dao类（MVC）

    （2）在service调用dao里面的方法

    //1. 原始方法
UserDao userDao = new UserDaoImpl(); //多态
userDao.update();  //update()方法在UserDaoImpl中定义过了

    (3) 在spring配置文件中配置, 首先配置UserService对象，在其中创建UserDao属性，接着注入属 性值，因为该属性值为对象（UserDaoImpl)，所以用ref指向被调用的对象，即被创建的 UserDaoImpl对象的id。

    <!--service和dao对象创建-->
<bean id="userService" class="com.spring5.service.UserService">
    <!--ref属性：创建userDao对象bean标签id值-->
    <!--因为注入的属性值是对象，所以不能用value，要用ref指向被调用的对象-->
    <property name="UserDao" ref="userDaoImpl"></property>
</bean>

<bean id="userDaoImpl" class="com.spring5.dao.UserDaoImpl"></bean>
