什么是React？

React用于构建用户界面的JavaScript库。

React是专注于视图的构建，不是一个请求库，也不是一个打包工具，主要提供UI层面的解决方案。它并不是一个框架，并不能解决项目中的所有问题。

React三大特性：

  1. 声明式编程

    ​ 命令式编程：用代码告诉计算机去做什么。

    ​ 声明式编程：通过代码告诉计算机。你想要的是什么，让计算机想出如何去做。

  2. 组件化

    ​ React提供了一种全新的语法扩展JSX，讲渲染逻辑和ui逻辑结合在一起，这个结合体就被称为组 J 件。

  3. 一次学习，随处编写

    ​ 学会react后可以在很多地方使用react语法来写代码，比如使用react来编写web端的页面； react+react native来写手机客户端APP；react+react 360开发VR界面。

Create-React-App（CRA)

​ —-官方提供的脚手架工具。

创建一个create-react-app

（前提：

​ 1. 安装并会使用node，包括包管理工具npm或者yarn。

​ 2. 了解ES6语法。）

步骤：

  1. npx create-react-app 项目名

    ( 或者npm install –g create-react-app

    create-react-app 项目名 )

  1. 用vscode打开项目

    Package.jason中

    "scripts": &#123;
    "start": "react-scripts start",  //启动项目
    "build": "react-scripts build",  //打包构建项目
    "test": "react-scripts test",   //运行测试
    "eject": "react-scripts eject"  //暴露webpack配置
  &#125;

    React script是CRA提供的命令工具.

  1. index.js

    在index.js中先import：

    import React from 'react'

    react并不负责渲染，所以在浏览器端需要引入React Dom来做渲染。

    import ReactDom from 'react-dom'
  再调用React Dom上的render方法。render的方法接受两个参数，第一个参数可以传入react组件或者react元素。第二个参数传入一个DOM元素作为渲染器的根节点。（一般会在html页面里面写一个id为root的div元素作为根节点，一般只有一个root节点）



  ```js

ReactDom.render(element, document.getElementById(“root”));
```

  在index.js中写的html不是html，是JSX，react定义的一种语法，实际上是个对象。

  ```js
  例： const element = <h1>hello world!</h1>;
  ```
  1. 启动项目：

    npm start

  1. 配置VSCode debugger功能：

    在项目根目录中创建.vscode\launch.json文件，输入如下代码：

    &#123;
    "version": "0.2.0",
    "configurations": [
      &#123;
        "name": "Chrome",    //配置名称
        "type": "chrome",    //启动的浏览器
        "request": "launch",  //启动方式
        "url": "http://localhost:3000",    //启动浏览器后自动访问的地址
        "webRoot": "$&#123;workspaceRoot&#125;/src",   //源代码所在目录
        "sourceMapPathOverrides": &#123;
          "webpack:///src/*": "$&#123;webRoot&#125;/*"   //sourcemap所在目录
        &#125;
      &#125;
    ]
  &#125;

    （配置代码可以在create-react-app中的development下的editor setup中找到）

  1. 测试：

    配置好之后在index.js的第六行打个断点，然后进入Vscode的debug窗口，点击运行，如果代码在运行到第六行就卡住了，并且在编译器的左边出现了变量和调用栈，则说明配置成功了。

至此，本地开发环境就基本搭建好了。
